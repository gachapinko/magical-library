# まほうの図書館 — PLAN.md

## プロダクト概要

子供の読書習慣を楽しく育てる読書記録PWA。
「5冊読む → スタンプ1個 → スタンプを使ってごほうび」というシンプルなゲームループが核。

---

## ゲームループ（確定仕様）

```
5冊読む
  └→ スタンプ1個獲得（本トラックアニメーション → スタンプゲットポップアップ）
       └→ 本トラックリセット、また5冊へ
            └→ スタンプが手元に溜まる
                 └→ 親がスタンプを消費してごほうびを渡す
```

---

## スタンプ設計（確定仕様）

### カウント方式
- `stamp_count` = 累計スタンプ獲得数（リセットしない）
- `used_stamps` = 累計スタンプ消費数（リセットしない）
- `inHand = stamp_count - used_stamps` = 現在手持ちのスタンプ数

### スタンプカード表示
- 手持ちスタンプ（`inHand`個）をすべて表示
- 7個以上の場合は横スクロールで表示
- 各スタンプは `empty` / `earned` の2状態
- 累計消費数は「星バッジ（星画像 + 数字）」で別途表示

### スタンプ状態設計
各スタンプは以下の状態を持つ：
- `empty` — 未獲得（淡い星型）
- `earned` — 獲得済み（宝石っぽい星型、タップ可）
- `consuming` — 消費アニメーション中
- `consumed` — 消費完了（一瞬だけ、すぐemptyへ）

### スタンプ消費アニメーション（仕様）
1. **チェック表示**（0.12〜0.16s）— 右下に小さいチェックマーク「ぽんっ」と出現
2. **発光**（0.18〜0.24s）— 白〜薄ピンクのハイライト + glow
3. **縮小変換**（0.16〜0.22s）— `scale(1) → 0.72`、わずかに上に浮く
4. **飛ぶ**（0.38〜0.5s）— 星バッジ位置へ弧を描いて移動、`scale(0.72 → 0.4)`
5. **カウント加算** — 到達時に `used_stamps += 1`、数字と星が `scale(1 → 1.14 → 1)` でpop
6. **元枠更新** — `consumed → empty`、必要なら左詰め再配置（spring）

全体1個あたり0.7〜0.9秒。複数消費時はqueueで順番実行。

### NG
- 一瞬で消えて何が起きたかわからない演出
- 派手すぎる爆発パーティクル
- 5個同時に飛んで視認しづらい
- 消費後の並び替えが唐突
- カウントがいつ増えたかわからない

## 本トラック・アニメーション設計（確定仕様）

### 通常進捗アニメーション（1〜4冊目）
- 新しく読んだ本のみが `bookActivate`（1.5s）
- `scale(1 → 1.15 → 1)` + `brightness(1.6) + drop-shadow(#ffd6ff)`

### 5冊達成アニメーション
1. 5冊目が `bookActivate`（1.5s）
2. 全5冊に `fullCardGlow`（1.5s）— `scale(1.08)` + `brightness(1.7)` + 二重drop-shadow
3. スタンプゲットポップアップ表示

### 本タップアニメーション（閲覧時）
- `done` な本：`book-pop-done`（0.56s）— 跳ねて光る + スパークル3個
- `current` な本：`book-pop`（0.56s）— 軽く跳ねる
